# DSA Practice

Collaborative data structures & algorithms practice repo for our class.

## How it works

1. **One folder per person** — work only inside `<Topic>/<your-name>/`
2. **Any language** — name your file `linkedlist.<ext>` (e.g. `.py`, `.java`, `.cpp`)
3. **Homework** — each assignment is a [GitHub Issue](../../issues); comment on the issue when you're done
4. **Self-evaluate** — no auto-grading; run the manual test cases in each topic's README

## Repo structure

```
DSA-Practice/
├── LinkedLists/
│   ├── linkedLists.md      # detailed theory & reference
│   ├── README.md           # homework specs & test cases
│   └── <your-name>/
│       └── linkedlist.*
├── Arrays/                 # coming soon
└── .github/
    └── ISSUE_TEMPLATE/
```

## Getting started

```bash
git clone <repo-url>
cd DSA-Practice
mkdir -p LinkedLists/<your-name>
# implement, commit, push
```

## Marking homework done

Comment on the assigned GitHub Issue with:

1. Your folder path (e.g. `LinkedLists/nishanth/`)
2. Language used
3. Self-check: did you pass the manual tests? (yes/no)

## Rules

- Do not edit other people's folders
- Do not delete shared docs (`linkedLists.md`, topic READMEs)
- Keep commits small and descriptive
